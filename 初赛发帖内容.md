### 1. Demo 简介

**是什么**：FocusCat 专注猫是一款 Android 原生 App，用一只会因你的专注而开心的虚拟猫咪，温柔地帮你把注意力从分心应用里拉回来。

**面向谁**：
- 学生党：自习时总忍不住刷手机、需要外部约束力的人
- 自由职业者 / 远程办公者：缺乏监督环境、容易被短视频和社交软件吞噬时间的人
- 拖延症人群：知道该做什么，但就是无法开始的人

**主要功能**：

**① 约束应用 + 反思问答**
把容易让你分心的应用（微信、抖音、B 站等）加入约束列表，每次打开都会先弹出反思问答页——「你进入这个 app 是来做什么的？」「你今天的 todo list 还有哪些？」所有问题必答，无法跳过。答完才能进入应用，进入后每隔 N 分钟还会再次提醒（间隔可在 5-120 分钟自定义）。

**② 专注模式 + 猫咪养成**
点击「开始专注」进入全屏沉浸视频背景的专注模式，计时器浮在天空区域，每专注 30 分钟自动为猫咪赢得一份猫粮奖励。回到主界面后可以投喂猫咪，播放专属进食动画，好感度随投喂次数累积，解锁「初识之友 → 默契伙伴 → 灵魂伴侣」三个成就。

**③ 双引擎守护 + 链路诊断**
采用无障碍服务 + 前台轮询双引擎检测，兼容所有 Android ROM。设置页提供 6 项链路诊断（守护总开关、使用情况访问、悬浮窗、无障碍、电池优化、约束应用列表），一眼看清权限状态，并提供「去授权」快捷入口。

### 2. Demo 创作思路

**灵感来源**：
作为一个容易被手机分心的人，我试过 Forest、番茄 ToDo 等专注工具，但它们要么是「种树」这种抽象奖励，缺乏情感连接；要么是强制锁机，让人产生抗拒心理。我一直想要一种「温柔但有力量」的约束方式——不是粗暴地阻止你打开应用，而是在你打开的瞬间，用一个让你停下来的问题把你拉回现实。而猫咪是最好的陪伴意象：它不会责备你，但会因为你的专注而开心。

**想解决的问题**：
- **「打开就停不下来」**：用户打开抖音「就看 5 分钟」，结果 2 小时过去了。约束应用 + 反思问答在入口处制造「停顿」，让用户重新审视自己的行为。
- **「专注没有正向反馈」**：传统的番茄钟只给数字，缺乏情感激励。猫咪养成系统让每一次专注都有可见的回报（猫粮 → 投喂 → 好感度 → 成就）。
- **「无障碍服务在国内 ROM 上不稳定」**：很多专注类 App 在小米/华为/vivo 上无障碍服务会被杀，导致拦截失效。FocusCat 采用双引擎检测 + 链路诊断，让用户清楚知道守护是否真的在工作。

**为什么做这个方向**：
- **情感化设计 > 工具化设计**：工具类 App 留存率低，因为用户用完就走。养成系元素让用户为了猫咪而主动打开 App，形成正向循环。
- **Android 原生 > 跨平台**：要实现应用拦截、无障碍服务、悬浮窗等功能，必须用 Android 原生能力，跨平台框架做不到。
- **学习工作赛道**：专注力管理是学习和工作场景的高频刚需，且现有产品体验都有提升空间。

#### 💡 TRAE 使用心得

1. **「代码精准翻译」能力出色**：将 Android Compose 的 `Brush.radialGradient`、`AnimatedVisibility`、`setKeepContentOnPlayerReset` 等平台特定 API，TRAE 能精准翻译为 CSS `radial-gradient`、CSS transition、视频事件监听等 Web 等价实现，保留了所有视觉与交互细节。

2. **长对话上下文管理**：38 个 Session 跨越 20 天，TRAE 的 memory 系统自动沉淀了项目硬约束（配色方案、圆角规范、视频边缘柔化规则等），避免重复沟通。

3. **Bug 诊断效率高**：双引擎检测的 Recurring quiz 重复弹窗问题，TRAE 通过分析 `onAppEntered`/`clearAppEnterSession` 生命周期，快速定位到 `lastAppEnterPkg` guard 缺失的根因。

### 5. 经验总结

- **情感化设计是留存关键**：猫咪养成系统让用户为了「再投喂一次」而主动专注，这种正向循环比单纯的锁机惩罚更可持续。
- **Android ROM 兼容性是深坑**：单靠无障碍服务不可靠，双引擎 + 链路诊断让用户自己也能排查问题，降低客服成本。
- **视频资源要提前规划**：4 种 idle 子动画 + eating + focus 共 12 个视频文件，命名规范和首帧 PNG 占位策略必须在开发初期就定好。
